## 功能概述

招聘端自动回复功能帮助招聘者自动化处理求职者的消息，支持：

- **自动监控消息** - 定时扫描新消息
- **智能筛选候选人** - 根据学历、工作年限、技能等条件筛选
- **快捷回复** - 预设消息模板，一键发送
- **LLM智能回复** - 使用AI生成针对性回复
- **多轮对话管理** - 追踪对话轮次，超时自动转人工
- **反检测策略** - 随机延迟、频率限制，模拟人类行为

## 快速开始

### 1. 启动应用

```bash
# 进入 UI 包目录
cd packages/ui

# 开发模式启动
pnpm dev

# 或构建生产版本
pnpm build
pnpm start
```

### 2. 登录 BOSS 直聘

首次使用需要登录 BOSS 直聘招聘者账号：

1. 点击左侧导航栏的「编辑登录凭据」
2. 按提示完成登录
3. 登录成功后关闭登录窗口

### 3. 配置浏览器

首次运行需要配置浏览器路径：

1. 点击左侧导航栏的「编辑浏览器偏好」
2. 选择自动检测或手动指定 Chromium 浏览器路径
3. 保存配置

### 4. 开始使用

1. 点击左侧导航栏的「招聘端自动回复」
2. 配置筛选条件和回复模板
3. 点击「保存并开始」

## 配置说明

### 基础设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 扫描间隔(秒) | 每次扫描消息列表的时间间隔 | 5 |
| 自动发送 | 是否自动发送回复，关闭则需要手动确认 | 关闭 |
| 发送前确认 | 自动发送关闭时，是否弹出确认对话框 | 开启 |

### 候选人筛选

启用筛选后，系统会根据条件自动过滤候选人：

| 筛选条件 | 说明 |
|----------|------|
| 学历要求 | 勾选接受的学历（大专/本科/硕士/博士） |
| 工作年限 | 设置最小和最大工作年限 |
| 期望职位关键词 | 候选人期望职位包含的关键词 |
| 技能关键词 | 候选人技能标签包含的关键词 |
| 屏蔽关键词 | 简历中包含则直接排除 |

### 快捷回复

预设常用回复模板，支持：

- 自定义回复名称和内容
- 启用/禁用单个模板
- 一键快速发送

默认模板：

| 名称 | 内容 |
|------|------|
| 收到简历 | 您好，收到您的简历，我们会尽快查看并给您回复，谢谢！ |
| 邀请面试 | 您好，您的简历已通过初步筛选，方便安排面试吗？请问您什么时间方便？ |
| 索要简历 | 您好，能发一份详细的简历吗？我们想进一步了解您的背景。 |
| 询问期望薪资 | 您好，请问您的期望薪资是多少？我们这边好评估一下。 |

### 回复策略

#### 匹配时回复

| 模式 | 说明 |
|------|------|
| 固定内容 | 发送预设的固定文本 |
| 第一条快捷回复 | 使用快捷回复列表中第一条启用的模板 |
| 随机快捷回复 | 从启用的快捷回复中随机选择一条 |

#### 不匹配时操作

| 操作 | 说明 |
|------|------|
| 跳过 | 不做任何处理，等待下次扫描 |
| 标记不合适 | 自动标记候选人为不合适 |
| 回复后跳过 | 发送婉拒消息后跳过 |

### LLM 智能回复（可选）

启用 LLM 智能回复需要先配置大语言模型：

1. 点击左侧导航栏的「配置大语言模型」
2. 添加 LLM API 配置：
   - API 地址（如 OpenAI 兼容接口）
   - API Key
   - 模型名称
3. 保存配置

支持的 LLM 功能：

- **首次回复生成** - 根据职位信息生成个性化的首次回复
- **后续回复生成** - 根据对话上下文生成针对性回复
- **面试邀约生成** - 自动生成面试邀约消息

## 使用指南

### 场景一：简单自动回复

适用于只需要简单确认收到消息的场景。

1. 关闭「候选人筛选」
2. 设置「固定内容」回复：`您好，收到您的消息，我会尽快回复您。`
3. 开启「自动发送」
4. 点击「保存并开始」

### 场景二：筛选 + 快捷回复

适用于有明确招聘要求，需要快速筛选的场景。

1. 开启「候选人筛选」
2. 设置学历要求：本科、硕士
3. 设置工作年限：3-5 年
4. 添加技能关键词：React、TypeScript、Node.js
5. 匹配时选择「随机快捷回复」
6. 不匹配时选择「跳过」
7. 点击「保存并开始」

### 场景三：智能对话

适用于需要深入沟通的场景。

1. 配置 LLM API
2. 在回复策略中选择「LLM智能回复」
3. 设置对话管理参数：
   - 最大对话轮次：3
   - 超过轮次自动转人工
4. 点击「保存并开始」

### 一键发送功能

运行任务后，可以随时使用「一键发送」功能：

1. 在 BOSS 直聘聊天页面选中候选人
2. 点击预设的快捷回复按钮
3. 消息自动发送

## 常见问题

### Q: 为什么启动后没有反应？

**A:** 请检查：
1. 是否已完成 BOSS 直聘登录
2. 是否已配置浏览器路径
3. 查看控制台日志是否有错误信息

### Q: 为什么消息没有发送？

**A:** 可能原因：
1. 「自动发送」关闭且「发送前确认」也关闭
2. 触发频率限制（每分钟/每小时操作上限）
3. 候选人已被筛选过滤

### Q: 如何避免被平台检测？

**A:** 系统内置了反检测策略：
- 随机延迟发送（3-10秒）
- 每分钟最多 10 次操作
- 每小时最多 50 次操作
- 每日最多 100 次操作
- 模拟真实打字行为

### Q: LLM 回复不准确怎么办？

**A:** 建议：
1. 使用更强大的模型（如 GPT-4、Claude）
2. 在快捷回复中预设更具体的模板
3. 调整回复策略为「固定内容」

### Q: 如何查看处理记录？

**A:** 系统会记录：
- 候选人对话状态（待处理/已匹配/已拒绝/已转人工）
- 每次回复的内容
- 筛选结果和原因

### Q: 如何停止任务？

**A:**
1. 在运行界面点击「结束任务」按钮
2. 或关闭应用窗口

## 配置文件

配置保存在 `boss.json` 文件中：

```json
{
  "recruiterAutoReply": {
    "scanIntervalSeconds": 5,
    "autoSend": false,
    "confirmBeforeSend": true,
    "constantReplyContent": "您好，收到您的消息，我会尽快回复您。",
    "candidateFilter": {
      "enabled": false,
      "degreeList": ["本科", "硕士", "博士"],
      "minWorkYears": 0,
      "maxWorkYears": 0,
      "skillKeywords": []
    },
    "quickReply": {
      "list": [...]
    },
    "replyStrategy": {
      "matchReplyMode": "constant",
      "notMatchAction": "skip"
    }
  }
}
```
